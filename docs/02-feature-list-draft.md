# 🧩 기능 리스트 초안 (Feature List Draft)

> 행사 신청 관리 플랫폼 — 주최자 중심 MVP
> 기반: docs/01-problem-definition.md

## 1. 주최자 핵심 플로우 (Core Flow)

주최자가 이 플랫폼으로 "신청을 받고, 명단을 손 안 대고도 정확하게 유지한다"를 완료하는 흐름.

1. 주최자가 행사 정보를 입력하고 신청 폼을 만든다 → 신청받을 준비 완료
2. 주최자가 신청 링크를 공유한다 → 참가자가 들어와 신청함
3. 참가자가 스스로 신청을 수정/취소한다 → 그 변경이 주최자 명단에 자동 반영됨
4. 주최자가 명단을 확인한다 → 별도 문의 응대 없이 현재 신청 현황을 한눈에 봄
5. 주최자가 명단을 내보낸다 → 현장에서 쓸 수 있는 최종 참가자 명단 확보

## 2. 기능 후보 10개 (Feature Candidates)

| # | 기능 이름 | 해결하는 문제 | 필요한 플로우 단계 |
|---|---|---|---|
| 1 | 행사 만들기 (제목·일시·장소·정원) | 매번 폼을 밑바닥부터 세팅하는 번거로움 | 1단계 |
| 2 | 신청 항목 설정 (이름·연락처 등 필수 입력) | 받을 정보를 행사마다 다시 구성 | 1단계 |
| 3 | 신청 링크 생성·공유 | 만든 폼을 채널에 뿌리고 전달 확인 | 2단계 |
| 4 | 참가자 신청 접수 | 신청을 받아 명단에 저장 | 2단계 |
| 5 | 참가자용 '내 신청 확인 페이지' | 참가자가 자기 신청을 다시 볼 창구가 없음 | 3단계 |
| 6 | 참가자 셀프 수정/취소 | '수정/취소해주세요' 문의가 주최자에게 몰림 | 3단계 |
| 7 | 명단 자동 반영 (수정/취소 즉시 동기화) | 수동 처리 누락으로 명단과 실제가 엇갈림 | 3·4단계 |
| 8 | 주최자 신청 현황 대시보드 (목록·인원수) | 현재 신청 상태를 한눈에 못 봄 | 4단계 |
| 9 | 정원/마감 표시 (마감 시 신청 차단) | 정원 초과를 수동으로 관리 | 1·2·4단계 |
| 10 | 명단 CSV 내보내기 | 현장에서 쓸 최종 명단 확보 | 5단계 |
