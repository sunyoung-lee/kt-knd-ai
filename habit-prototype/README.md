# 습관 트래커 — 학습용 프로토타입 (강사용 레퍼런스)

"AI 활용 PM 한 사이클(기획→배포)"을 **9시간**에 경험하기 위한 예제 제품입니다.
빌드 도구 없이 **정적 HTML/CSS/JS + localStorage** 만 사용해, *도구가 아니라 제품 사고*에 집중하도록 설계했습니다.

---

## 1. 파일 구조 (관심사의 분리)

| 파일 | 역할 | 핵심 학습 포인트 |
|------|------|-----------------|
| `index.html` | 구조 | 시맨틱 마크업, CSS/JS 연결 위치 |
| `style.css` | 디자인 | CSS 변수, Grid로 줄맞춤, `.show` 토글로 모달 |
| `app.js` | 동작 | 상태(state) 한 곳 관리, `save() → render()` 흐름 |
| `README.md` | 가이드 | 본 문서 |

> **한 줄 요약:** 상태는 `habits` 배열 하나. *상태가 바뀌면 → 저장 → 다시 그린다* 만 반복합니다.

## 2. 동작 원리 한눈에

```
사용자 클릭
   │
   ▼
상태 변경 (habits 배열 수정)
   │
   ├─ saveHabits()  → localStorage 에 저장 (새로고침해도 유지)
   └─ render()      → 화면 전체를 현재 상태로 다시 그림
```

## 3. 로컬 실행 / 배포

- **실행:** `index.html` 더블클릭 → 브라우저에서 바로 동작 (서버·설치 불필요)
- **배포:** 저장소를 GitHub에 올리고 → GitHub 연동 자동배포(Vercel/Pages) 연결 → 이후 `git push` = 자동 배포

---

## 4. 9시간 커리큘럼 흐름 (기획 → 배포)

> 원칙: **도구는 3개로 고정** — AI 코파일럿(Claude Code) · 저장소(GitHub) · 배포(자동배포). 그 외 추가 금지.

| 시간 | 단계 | PM이 결정하는 것 | AI에 위임 | 산출물 |
|------|------|-----------------|----------|--------|
| 0:00–0:30 | 환경 세팅 | (1회만) | — | 빈 라이브 URL |
| 0:30–2:00 | **기획** | 문제·타깃·핵심가치 | PRD/유저스토리 초안 | PRD 1장 |
| 2:00–3:00 | **설계** | MVP 범위(뭘 뺄지) | 화면·플로우 초안 | 화면 흐름도 |
| 3:00–3:30 | 버퍼 | — | — | — |
| 3:30–6:00 | **구현** | 요구사항·검수·피드백 | 프로토타입 코드 | 동작 프로토타입 |
| 6:00–7:00 | **배포** | push→배포 확인 | 배포 오류 수정 | **라이브 URL** |
| 7:00–8:00 | **검증** | 직접+동료 테스트 | 개선 1사이클 | 피드백→수정 커밋 |
| 8:00–8:45 | **회고** | 위임/판단 기록 | — | 회고 노트 |
| 8:45–9:00 | 공유 | 데모 발표 | — | 발표 |

### 함몰 방지 규칙 (참가자 공지용)
1. **30분 룰** — 한 버그에 30분 막히면 AI에 통째로 위임하거나 **범위를 줄인다**.
2. **완벽 금지, 완주 우선** — 기획→배포 *한 바퀴*가 목표. 기능은 1~2개로 충분.
3. **결정을 산출물로 남긴다** — 코드보다 PRD·범위결정·회고가 진짜 결과물.
4. **새 도구 추가 금지** — "이거 써보면 좋을 듯"이 함몰의 시작.

## 5. 확장 과제 (학습자용 — 난이도 순)

1. **🟢 쉬움:** 이모지 목록(`EMOJIS`)에 새 아이콘 추가하기
2. **🟢 쉬움:** 색 테마 바꾸기 — `style.css` 의 `:root` 변수만 수정
3. **🟡 보통:** "오늘 모두 완료" 시 축하 메시지 띄우기
4. **🟡 보통:** 습관별 **누적 달성 횟수** 표시 (`Object.keys(habit.done).length`)
5. **🔴 도전:** 7일 → **월간 달력** 보기로 확장
6. **🔴 도전:** 데이터 **내보내기/가져오기**(JSON 파일) 기능

각 과제는 "PM이 요구사항을 적고 → AI에게 구현을 맡기고 → 결과를 검수"하는 식으로 진행하면
9시간 사이클의 *축소판*을 반복 연습할 수 있습니다.
