# 문제 정의서 초안

> 행사 신청 관리 플랫폼 — PM 인터뷰 기반 초안
> 태그: ✅ PM이 직접 말한 표현 · ⚠️ AI가 해석·재구성 · ❌ AI가 전문가 판단으로 추가

## 페르소나

**A. 주최자**
- 누구: 행사를 여는 사람 [✅] — 인터뷰 설정에서 PM이 정의한 페르소나
- 핵심 고통: 신청 후 문의 응대, 특히 '수정/취소해주세요' 요청이 몰려 명단과 실제가 엇갈림 [✅] — 주최자 ①·②·③에서 PM이 직접 고른 표현
- 현재 해결 방식: 구글폼 + 시트 조합으로 받고, 수정/취소는 수동으로 시트에 반영 [⚠️] — '구글폼+시트'는 ✅, '수동 반영'은 '명단 엇갈림'에서 추론한 보충
- 원하는 것: 수정/취소 문의에 끌려다니지 않고 정확한 명단을 유지하는 것 [⚠️] — '문의 응대 폭주'·'명단 엇갈림'에서 역으로 추론한 바람

**B. 참가자**
- 누구: 신청하는 사람 [✅] — 인터뷰 설정에서 PM이 정의한 페르소나
- 핵심 고통: 스스로 수정/취소를 못 해 생기는 연락해야 되는 부담감 [✅] — 참가자 ①·②에서 PM이 직접 고른 표현
- 현재 해결 방식: 주최자에게 직접 연락해 수정/취소를 부탁함 [⚠️] — '연락해야 되는 부담감'에 내포된 방식을 풀어 쓴 해석
- 원하는 것: 내 신청 확인 페이지 [✅] — 참가자 ③에서 PM이 직접 고른 표현

## 문제 정의

- 사용자: 행사를 여는 주최자와 신청하는 참가자, 양쪽 모두(마켓플레이스) [✅] — 핵심 사용자로 PM이 선택한 표현
- 핵심 문제: 참가자가 스스로 수정/취소를 못 해 '수정/취소해주세요' 문의가 주최자에게 몰리는, 신청 후 확인·변경의 번거로움 [✅] — '신청 과정의 번거로움'·'신청 후 확인/변경'·주최자/참가자 답변 조합
- 현재 맥락: 주최자가 구글폼 + 시트 조합으로 운영하며 신청 후 문의를 처리하는 상황 [✅] — '구글폼+시트', '신청 후 문의 응대' 모두 PM 선택 표현
- 기대 결과: 참가자가 내 신청 확인 페이지에서 셀프로 처리해 문의 응대 폭주가 줄어듦 [⚠️] — '내 신청 확인 페이지'·'문의 응대 폭주'는 ✅, '줄어든다'는 효과 예측은 해석
- MVP 우선 페르소나: 두 페르소나를 잇는 한 줄기 [✅] — 7번 우선순위 질문에서 PM이 직접 고른 표현

## 시니어 PM 추가 분석 (인터뷰 외 AI 판단 — 전부 ❌)

- JTBD [❌] — "행사에 신청했을 때, 주최자에게 연락하지 않고도 내 신청 상태를 확인·변경하고 싶다. 그래서 안심하고 참석을 준비하고 싶다." 인터뷰에 JTBD 진술은 없었음.
- 핵심 성공지표(North Star 후보) [❌] — '행사당 수정/취소 관련 문의 건수'. 이 한 지표가 양쪽 고통(주최자 응대·참가자 부담)을 동시에 대변. 보조: 셀프 수정/취소 완료율, 명단 정확도(현장 인원 일치율).
- 경쟁/대체재 [❌] — 구글폼+시트(현 대안), 네이버폼/타입폼(폼 도구, 셀프변경 약함), 이벤터스·온오프믹스(행사 플랫폼, 범용·무거움), 페이스북/카톡 이벤트(소통 중심). 빈틈: '참가자 셀프 수정/취소 ↔ 명단 실시간 일치'를 가볍게 제공하는 도구가 부재.
- 주요 리스크 [❌]
  - 인증 부재 리스크: 로그인 없이 URL로 신청을 여는 MVP 특성상, 링크 유출 시 타인이 남의 신청을 보거나 취소할 수 있음 → 토큰화된 고유 링크 필요.
  - 동시성 리스크: 정원/마감이 있는 행사에서 셀프 취소→대기자 승계가 자동화되지 않으면 '명단 엇갈림'이 형태만 바뀌어 재발.
  - 채택 리스크: 주최자가 이미 구글폼 무료에 익숙 → 마이그레이션 비용을 넘는 '명단 자동 일치' 가치가 첫 사용에서 즉시 체감돼야 함.
- MVP 가설(검증 대상) [❌] — "참가자에게 셀프 수정/취소가 가능한 '내 신청 확인 페이지'를 주면, 주최자에게 오는 수정/취소 문의가 유의미하게 줄고 명단 정확도가 오른다."
