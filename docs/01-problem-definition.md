# 문제 정의서 (확정본)

- **사용자**: 행사를 직접 여는 주최자와 신청하는 참가자를 잇는 양면(마켓플레이스) 사용자이며, 그중 운영 부담이 더 큰 주최자가 우선 대상이다.

- **문제**: 신청 자체보다 '신청 후 확인/변경' 단계가 자동화되지 않아, 주최자에게 '문의 응대 폭주'가 발생하는 신청 과정의 번거로움이 핵심 문제다.

- **맥락**: 주최자가 구글폼+시트 조합으로 행사를 운영하면서, 접수는 되지만 확인·리마인더 발신이 안 돼 모든 후속 소통을 카톡·문자로 일일이 수동 응대하는 상황이다.

- **기대 결과**: 참가자가 스스로 자기 신청을 확인·변경·취소할 수 있게 되어, 주최자가 카톡·문자 응대 없이 행사 운영에만 집중할 수 있게 된다.
